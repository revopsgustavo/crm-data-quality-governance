# Contexto do Case

Empresa fictícia SaaS B2B com operação comercial baseada em leads, contas, contatos, oportunidades, atividades, forecast category, close date e tarefas de remediação.

O objetivo é mostrar como RevOps transforma CRM Data Quality em Revenue Governance. Campo vazio, duplicidade ou forecast inconsistente não são apenas falhas técnicas: são riscos para forecast, pipeline review, accountability comercial, produtividade e decisões executivas.
